java►Class的学习

java中的Class类

学一下子 Class对象
1、java有个类对象，是一个类 ，名字是Class，封装类型的类（好好体会这句话）
2、JVM在加载某一个类或者接口，比如是MyClass的时候，会自动创建一个Class类的对象（ClassLoader生成），这个对象代表了MyClass类或接口的抽象表示（类型）
3、Class这个类的实例呢，是JVM根据咱们自己写的class（Myclass）建立这个类的Class实例
就是说，Class和Myclass是平级的，Class的每个实例一一对应着咱们自己写的类，通过Class实例可以创建某个类的对象，也可以通过某个类找到JVM创建的这个类的Class实例

我现在自己写了一个Myclass.java

如何获得JVM创建的这个类的Class对象呢，有三种方法

1）每个类都隐藏了一个静态变量class，该变量就是存放这个类的Class实例
Class class1 = Myclass.class;
2）通过getter方法获取1）中的静态实例
Class class2 = my.getClass();
3）通过Class的静态方法forName

总结：每个类相应的Class对象是独一无二的，是类加载器生成
加载某个类进内存的时候会初始化他的静态块，创建某个类的实例的时候，首先要看他是否在内存中，不在的话，先加载进来，再初始化他的非静态块，生成一个实例。
所以，若只在程序中有Class child = Class.forName(“Child”);

如何通过这个Class实例创建相应的类的对象呢

Class class1 = Myclass.class;//先获取这个类的Class对象
Myclass myclass2 = (Mycalss)class1.newInstance();//通过Class对象的newInstance()方法
注意：这种方法的使用前提是 Myclass这个类要有一个无参构造函数，否则代码执行时会直接crash掉，因为newInstance方法会调用类Myclass的无参构造方法。而java又是如果显示定义了有参构造函数，那么就不会提供默认的无参构造函数，你还得自己写一个无参的。

实验的时候发现经过javac编译成.class文件后，才会生成这个类的Class对象，因为，如果我没有编译这个类，在Class.forName(“Child”);获取Class实例的时候会抛出异常ClassNotFounException，编译有.class文件后就可以获得了，所以应该是类加载器根据.class文件生成这个类的Class对象。加载类进入内存的时候，会初始化类的静态块，而且是先初始化父类的，再初始化子类的。
而在用B b = new B(); Class bb = b.getClass();的时候

MySql►权限控制

Mysql权限控制

今天连接一个数据库的时候，突然对使用的Navicat有点好奇，以前只知道建个数据库，今天来了解一下

一般就这么建立一个远程连接，或者一个本地连接，我现在的理解是，每个连接里面可以建立多个数据库，本地连接对应自己本机的数据库，可以建立多个本地连接，但是端口号一样，主机名也一样，你操作的实际上是一个东西，我尝试建立了一个tengyun本地连接和mysql0107本地连接，在任意一个里面新建一个数据库，并建立表格，在另一个连接里都会显示出来。好像这么简单的一个事我还写了这么多字。

先看下怎么创建用户吧

首先，我们在安装mysql的时候，就已经创建了root用户，也可以在那时候创建其他的用户，由于我已经安装过了，这里偷偷放一张网上的图片

首先你要以root用户登录吧，在navicat里面建立一个连接的时候，就已经包含了登录的用户的信息，如果是在自己的命令提示符里面登录MySql的话，就是 mysql -u root -p回车然后输入你的root用户密码

登录之后开始以root用户的身份创建其他用户
命令：create user ‘1’@’2’ identified by ‘3’
这里面
1 是你创建的用户名
2 是创建的用户可以在哪个主机上登录，本地的就是localhost，也可以是某个ip地址，如果想让这个用户从任意远程主机登录，可以使用通配符%来匹配
3 是这个用户的登录密码，为空的话就不需要登录密码了
比如说我现在创建一个tengyun用户，允许在任何地方远程登录，密码是tengyun吧
create user ‘tengyun’@’%’ identified by ‘tengyun’; （注意一定要有分号作为结束符）

在一个连接里面，以root身份登录之后，有四个基本数据库

这四个，据说是存放这个连接的一些基本配置，比如这个连接有哪些用户，在mysql数据库的user表里面，存放了用户名、以及该用户的所有权限还有密码什么的





你看目前我的这个本地连接有root用户，还有两个我现在不认识，看列名，priv，就是privilege，权限，的缩写了，所以这每一列大多都表示一种权限

MySql中的用户权限管理有：

1、某用户可以访问哪些库和表
2、某用户可以对哪些表执行select、create、delete、alert等等操作
3、某用户可以在哪里登录
4、某用户是否可以对别的用户授权

看下授权的命令格式
grant 1 on 2.3 to ‘4’@’5’ identified by ‘6’ with grant option
1 是权限名称，可以是all privileges、create、index等等好多权限的名称，可以看mysql的官网https://dev.mysql.com/doc/refman/5.7/en/privileges-provided.html ，下面我把那个表放这里了
2 和 3 就是你授权的用户可以访问哪些库的哪些表 比如zhy库里的user表，就是zhy.user，如果可以访问所有库里的所有表，用*，来表示，就是*.*
‘4’@’5’ 就是被授权的用户，用户名已经该用户可以在哪里登录 例如’tengyun’@’%’ 表示tengyun这个用户可以在任何地方登录
6 就是被授权的这个用户的密码
with grant option 表示被授权的用户可以将自己的权限授予其他用户

所以你要是想给某个用户授权，你还得知道这个用户的一些信息才行。

MySql为了安全，用户默认只允许在本地登录，但是要远程登录的时候，你就要给他授权了。
记住对所有用户的权限做了操作之后都要刷新权限，将权限信息从内存写进数据库。 flush privileges;这里面privileges不是表，应该就是一个命令吧。

对root用户的授权

1、一般，我们需要允许root用户在任何地方都可以远程登录，并具有任何数据库的任何操作权限
怎么设置呢？
需要在本机先使用root登录mysql —> mysql -u root -p “yourPassword”;
然后授权 —> grant all privileges on . to ‘root’@’%’ identified by ‘yourPassword’ with grant option;
最后重载授权表 —> flush privilege;
退出吧 exit
这样的话，就是自己给自己授权吧，如果说，你买了个服务器，把mysql装那上面了，那怎么在本机用root登录呢？你只能是远程的，据说可以使用linux来搭建，目前这里还没搞清楚，等我再请教一下再来写写。所以这里的问题是：如何将mysql数据库部署在阿里云上？

2、设置root用户可以在一个特定的IP进行远程的登录，并具有所有库的所有权限
重复1中的步骤，只需要在授权的地方改一下
grant all privileges on . to ‘root’@’192.45.67.89’ identified by ‘yourPassword’ with grant option;
如果说不是授予所有的权限，那就改成
grant select,insert,update,delete on . to ‘root’@’192.45.67.89’ identified by ‘yourPassword’ with grant option;

查看某个用户的权限

首先，你还是要以某个用户的身份登录，然后再去查看某个用户的权限
show grants

删除用户的权限

这个需要revoke命令 按照1和2里面，root先登录mysql
比如咱们先授个权吧 grant select，insert，update，delete on someDB.* to ‘someUser’@’%’ identified by ‘password’;
删除someUser的所有权限：revoke all on someDB from someUser; 注意了，该操作只是清除了tengyun用户的权限，但是这个‘tengyun’用户还是存在的
重载授权表： flush privileges;
要是想删除这个用户，那就删了吧 delete from mysql.user where user = ‘tengyun’;
退出mysql数据库： exit

可以使用 select host,user from mysql.user 来查看mysql有哪些用户，以及他们可以登录的host

4、删除用户
除了上面的delete 还有drop可以用
drop user ‘tengyun’@’%’;

查看某个用户有哪些权限

show grants for ‘tengyun’@’%’;

权限有哪些呢？

全局管理权限：
1、file: 在MySQL服务器上读写文件。
2、process: 显示或杀死属于其它用户的服务线程。
3、reload: 重载访问控制表，刷新日志等。
4、shutdown: 关闭MySQL服务。

数据库/数据表/数据列权限：
1、alert: 修改已存在的数据表(例如增加/删除列)和索引。
2、create: 建立新的数据库或数据表。
3、delete: 删除表的记录。
4、drop: 删除数据表或数据库。
5、index: 建立或删除索引。
6、insert: 增加表的记录。
7、select: 显示/搜索表的记录。
8、update: 修改表中已存在的记录。

特别的权限：
1、all privileges: 允许做任何事(和root一样)，这里面privileges可写可不写
2、usage: 只允许登录，其它什么也不允许做

算法►排序算法

各种排序

周四面试的时候，有排序的算法，唉，以前都没有注意，朋友来找我玩了几天，今天有空来总结一下。

实现一组无序数的升序排列，假设是10个整型数，a[10] = {34,56,23,1,54,76,34,53,27,9}

冒泡排序 bubble

时间复杂度：O(n²)
实现描述：每次将最大的数挪到未排序的所有数的最右边，从而实现排序

注意：双层for循环的范围要搞清楚

选择排序 select

算法描述：这个数组分有序区和无序区，初始状态是整个数组都是无序区，然后每次无序区选择一个最小的数将他放在有序区的最后边，末状态是整个数组都是有序区

把握好无序区和有序区即可

快速排序 Quick

java中Arrays类中的有关sort的排序用的就是快速排序，是目前已知的最快的排序方法，是冒泡排序的一种升级版
描述：通过一趟排序，将整个数列划分成两组，一组中的所有数要比另一组的数小，然后对每一组，再这样递归，最终形成一个有序数列

注意：一定要把算法的过程搞清楚，然后就是基数的选择，可以任意选择，也可以就选最左边的数。递归程序要想清楚，以后多过来看看想想。如何做到遍历一次就让基数的左边都比基数小，基数右边的数都比基数大，这个应该就是算法的精髓了。

插入排序 Insert

描述：也分有序和无序部分，就是将每个元素，插入有序部分中，最终形成整体有序

注意，在寻找插入位置时，有两种情况，一种是有序区的最左边的数比要插入的数大，一种是在有序区的中间位置找到比要插入的数小，这两种情况细枝末节不太一样，要分开考虑。

归并排序 Merge

描述：把一组分成两组，每一小组再分为两组，就这样一直递归分下去，最终呢，如果只有两个数，就按序排好返回去，只有一个不用排就返回去。这是末端情况，中间情况就是，得到了两个部分的排序后，将这两个小组排好序返回。这里排序呢，要注意一下，就是两个有序数列的合并。

Linux►文件系统

Linux的文件系统

Linux下的文件系统

什么是文件系统？来理解一下

文件系统是基于被划分的存储设备上的一种文件的命名、存储、组织及读取的方法。

Linux文件系统的工作原理

文件系统通过为每个文件分配文件块的方式把数据存储在硬盘中。所以要维护每个文件的所有文件块的分配信息，这些信息也都存储在硬盘上。不同的文件系统，它的分配方式和读取文件块的方式是不一样的。
文件的块结构信息也称为meta-data（元信息），包括文件的权限、所有者、创建和访问和更改的时间等等这些与文件有关的信息。

数据结构►算法概念

算法和算法的评价

什么是算法

算法是对问题的求解的一种描述。通俗说，就是解决一个问题的方案。

算法的5个重要特性：有穷性、确定性、可行性、输入、输出
1、有穷性：一个算法在执行有限的步数之后一定要停止
2、确定性：算法的每一条指令必须要有明确的含义。也就是说，相同的输入必须得到相同的输出
3、可行性：算法中的每个操作都必须是基本的运算执行有限次就可完成的
4、输入：一个算法有m(m >= 0)个输入
5、输出：一个算法有n(n >= 1)个输出
