mysql安装教程

下载安装解压缩版MySql

1、下载

访问网址https://dev.mysql.com/downloads/mysql/ 下载相应版本的.zip文件

解压到要安装的文件夹下，例如D:\Tools\develop\mysql

2、写配置文件

在安装目录下新建my.ini文件，用编辑器打开,书写配置文件

3、配置环境变量

在环境变量的系统变量中，双击系统变量中的Path，在这后面添加mysql安装目录的bin目录，例如添加D:\Tools\develop\mysql\mysql-5.7.23-winx64\bin，点击保存就好了。

4、安装和初始化

1）在cmd命令行，切换到mysql的bin目录下

2）输入 mysqld -install 回车后出现安装成功的字样
3）输入 mysqld –initialize （注意这里是两个短横杠）
4) 输入 net start mysql （启动mysql服务）
5）输入 mysql -u root -p 进入mysql服务命令行模式更改root用户的密码。
6）输入 user mysql; (注意输入分号后回车)
7）输入update user set authentication_string =password(“tengyun”) where user=”root”;

5、重新开启mysql服务

1）在控制面板中搜索服务，找到mysql服务，点击关闭服务
2）打开安装目录的my.ini文件，将最后一句skip_grant_tables用中括号括起来，表示注释掉
3）在CMD命令行，输入 net start mysql，重新启动mysql

结束！

PS：
若重启服务失败，删除安装目录的data文件夹，重新输入命令

Python的赋值

首先要牢记，六类数据，分成两种，一种是原子型，Number和String，String又可以分解为字符。一种是容器型Set、Tuple、List、Dictionary。在赋值过程当中：

1、当值是原子型的，会先在内存中寻找是否有这个值，有的话就将这个值的地址直接赋给变量，没有的话，就开辟一个内存空间，将值放进去，再把地址赋给变量。甚至是单个字符也是这么做的。
2、通过多组实验，我认为，String、Set、Tuple、List、DIctionary，这些对象，每个对象会存储除了数据还有一些以外的东西，例如String对象的地址，和String里面每个元素的地址都是不一样，并且杂乱无章。


字符串一样的，首地址一样，字符一样的，地址是一样。字符串首地址和第一个字符地址不一样。字符串的字符地址杂乱无章。首地址应该是这个字符串对象的地址，这个对象存储了一些东西。


对于Tuple，如果内部仅包含Number、String、Tuple并且完全一样，那么Tuple的首地址是一样的（指向相同的对象），而且，内部每个元素地址也都一样，但是要是包含Set、List、Dictionary，那么首地址不一样.


。。。。。。重大发现，我要是在命令行窗口敲代码，两个tuple即使都是Number，他们的首地址也不会一样，是两个不一样的tuple对象，但是各个元素的地址还是一样的。。。。。。。。。好奇怪啊


对于Set，即使内部元素完全相同，首地址也不一样，说明是两个完全不同的对象。要记得，Set的元素只能是Number、String、Tuple这三种类型的对象


对于List，无论怎样，都会重新开辟内存分配空间创建新对象，但是对象内部的值的创建，还是和前面发现的一样。并且，当我改变一个元素的时候，系统会按照相应的原则去分配空间。例如这里面，系统就分配空间，值为9，在将地址给list2[0]，注意，地址改变了！以前的那个元素就找不到了！

同样的对于Dictionary来说，可以指定没有的key来增加键值对，当为存在的key指定值的时候，会遵循前面那些规律来分配！

还是先记录AMD规范。全称 Asynchronous Module Definition 异步模块定义，AMD规范是CommonJS组织提出的JavaScript的架构方案和标准。js是一门语言，使用js构建的框架比如jQuery、Dojo、ExtJs、requireJs。那么像AMD这种规范，就是那些框架按照规范的指引去实现。AMD规范，完整的描述了模块的定义、依赖、引用和加载。NodeJs是一门c++编写的框架，这个框架用来运行js代码。

3、后台用@RequestBody和@RequestParam，前后台数据的交互

这两个注解都是用于Controller方法中接受参数的注解。
1、@RequestParam
用于处理Content-Type是”application/X-www-form-urlencoded”编码的内容。当content-Type是这个的时候，get方式和Post方式的值都会被servlet接受并转化到Request.getParameter()参数集中，@RequestParam注解则是将Request.getParameter()中的Key-Value参数利用Spring的转化机制ConversionService，转化成参数接受对象或字段，所以这时候前台传的数据可以是对象也可以是简单类型的属性。在http协议中，Content-Type默认是”application/X-www-form-urlencoded”。所以我们可以就默认好了。

2、@RequestBody
与上面对应的，@RequestBody则是用来处理Content-Type非”application/X-www-form-urlencoded”编码格式的数据，并且@RequestBody用于处理HttpEntity传递的数据，所以Get请求不能用@RequestBody。在Post请求中，是HttpEntity来传递数据，这时候，就要在请求头中声明Content-type为”application/json”，并且要将传递的数据改为JSON或者XML格式，一定要是这两种形式。

所以前台，如果是定义一个js对象，那么可以用JSON.Stringify(data),转化为Json格式而不是键值对的格式，并且指定ContentType为application/json，后台再用@RequestBody就可以接受了。
