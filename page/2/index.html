SpringMvc►标签►RequestMapping

content-type的学习

首先我们要明白一个，这个标签作用于处理器上，是一个匹配请求和设置返回值的一个规则，请求是浏览器发出的。我是这么理解的。

一、@RequestMapping作用的地方

1、可以作用于类上，表示访问这个类的规则
2、作用于方法上，表示访问这个方法的规则

二、@RequestMapping有哪些参数

有六个参数 value、method、consumes、produces、params、headers
1、value：表示路径

2、method：表示浏览器请求的method类型，可以是POST、GET等等。如果你的表单请求是POST，那么你想访问的这里也要是POST才能匹配上。你在浏览器的一个a标签里请求一个方法，a标签的href默认是GET请求，那么请求的方法的这里只有设置为GET才能匹配成功，才能访问。

3、consumes：表示浏览器这次的请求提交的数据的内容类型（content-type），就是说我这个方法我接受的数据的内容类型，可以是application/json、text/html，比如说在页面中提交一个表单，content-type会默认为application/x-www-form-urlencoded，那么你在这边的方法中要是说你只接受@RequestMapping(consumes="application/json")，那就会出现415错误（由于媒介类型不被支持，服务器不会接受请求）。

4、produces：表示这个方法返回给浏览器的数据是以什么格式返回的，也就是告诉浏览器以这种格式去解析数据。只有当请求头（request header）中的accept包含这个类型时才能返回

再比如说这个请求头的accept： text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8
这里面*/*;q=0.8，表明可以接收任何类型的，权重系数0.8表明如果前面几种类型不能正常接收，则使用该项进行自动分析。application/json 几种主流浏览器都可以自动解析。

5、params：表示这个请求中一定要包含的参数

这样的话，就表明，请求过来的必须要有username和age这两个参数，并且age不能为10

6、headers：表示请求中一定要包含指定的header值

这个就是说，你的请求头中的accept-language这个参数的第一语言是US（这里有点没转过弯来）。

请求会先根据url找路径，这时就会根据请求头做出一系列的判断，匹配不通过，也就是找不到，就会返回404.找到了若果不能正常处理就会返回5XX的错误。

参考自 https://blog.csdn.net/J080624/article/details/55193269

SpringBoot

Springboot的学习

首先简单学习了Spring的配置Bean的发展，XML配置到注解配置，再到现在的Java配置。
1、JDK1.5带来了注解支持，使得Spring2.x开始使用注解对Bean进行申明和注入。注解配置，大大减少了XML文件，也简化了项目的开发。只需要在使用Bean的地方写上注解就可以了。这时，我们遵循，应用的基本配置（数据源，资源文件）用xml，业务开发（Service注入Bean）用注解
2、Spring3.x开始提供了Java配置方式，这能更好的理解咱们配置的Bean。Spring4.x 和 SpringBoot都推荐使用java配置方式

那来学习一下Java配置长什么样子

1、注解：@Configuration 和 @Bean
作用： @Configuration作用于类上，表示这个类是一个java配置文件
@Bean 作用于方法上，表示这个方法返回一个Bean
2、简单来个粒子

其实现在还是不太明白，配置到底是在配置什么，Spring的原理还是不太清楚。毕设在即，先速成一下吧。

读取外部的资源配置文件

使用的注解： @PropertySource 和 @Value
@PropertySource 用于指定读取的配置文件
@Value 用于获取值，就是value写上配置文件中的键，然后把值给某某某
比如说，我们把数据源的配置写在了application.properties里面，那么怎么写配置呢

什么是动态语言，什么是静态语言？
静态语言是先编译后运行，动态语言是无需编译，直接运行。静态语言，比如Java、C，动态语言，比如Ruby、Groovy、Scala、Node.js、JS等等

随着动态语言的流行，Java开发显得笨重，配置、效率、部署流程等等都存在很大的问题，SpringBoot的生成，就是为了克服这些，SpringBoot让我们很容易创建一个独立运行的准生产级别的基于Spring框架的项目，SpringBoot会提供默认的配置，会解决部分依赖问题，它还内嵌了容器，框架强大吧。

创建SpringBoot应用

pom里面
首先 parent必须是 spring-boot-starter-parent,这里面的parent包含了大量的默认配置，大大简化了我们的开发（原来是在这里面）。
然后 添加web支持，是 spring-boot-starter-web
然后 添加springboot的插件 spring-boot-maven-plugin

SpringBoot项目会有一个入口类 ****Application.java，这个文件里面，有一个main方法，是标准的java应用程序的入口方法。

SpringApplication用来从java main方法启动一个spring应用，默认的启动步骤如下：
1)创建一个合适的ApplicationContext实例,这个实例取决于classpath。
2)注册一个CommandLinePropertySource，以spring属性的形式来暴露命令行参数。
3)刷新ApplicationContext，加载所有的单例bean。
4)触发所有的命令行CommanLineRunner来执行bean。
大部分场景下，可以从你的application的main方法中直接调用它的run()静态方法。就是最后一句没有被注释掉的代码。其实这个人解释的这四部分，我不太理解，先放着。

这是个入口类，还可以作为配置类，还能是一个controller
先讲一下@SpringBootApplication这个注解
@SpringBootApplication是SpringBoot的核心注解，是一个组合注解，咱么可以按ctrl打开看一下

@SpringBootApplication = @Target + @Retention + @Document + @Inherited + @SpringBootConfiguration + @EnableAutoConfiguratio + @ComponentScan
这里面 比较不基础的，有@SpringBootConfiguration、@EnableAutoConfiguration、@ComponentScan
1、@SpringBootConfiguration = @Target + @Retention + @Documented + @Configuration

2、@EnableAutoConfiguration 表示开启自动配置，SpringBoot会根据你在pom中引入什么依赖，他就会智能的帮你配置这些引入的jar的配置

3、@ComponentScan 默认扫描@SpringBootApplication所在类的同级目录以及它的子目录

综合来说，@SpringBootApplication这个注解的作用就是开启自动配置的注解，如果，你不想让你的某个模块采用自动配置，就这样
@SpringBootApplication(exclude = {XXXX.class})

好了，如果，你想修改默认配置，那么在全局配置文件中，修改，application.properties就是这个全局配置文件，往往，我们在这里采用“选择某个配置文件”的方式来写，就是说，在application.properties中指明你现在用的全局配置文件是什么，这么写spring.profiles.active=dev,然后，你就采用的是application-dev.properties这个配置文件，还可以是spring.profiles.active=pro，表示你用的是application-pro.properties，配置文件的类型，也可以是其他的，比如说yml类型的。全局配置文件中的键的意义有官方的标准。
