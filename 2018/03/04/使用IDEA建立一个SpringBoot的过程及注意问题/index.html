使用IDEA建立一个SpringBoot的过程及注意问题

下面是简单的按照网上的教程建立的一个springboot的项目，过程简单，但是也有些地方需要注意，记录下来

1、新建项目的步骤，很简单，在网上搜下教程就有
Spring Initializer -> web web -> mysql jdbc mybatis ->还有个thymeleafs这个选上 -> 然后就是一些命名

2、由于项目新建后就会引入maven，所以在IDEA中配置一下全局设置（default settings中设置maven），这样的话就不会下载到c盘了

3、最重要的设置modules里面，别忘了Web这个选项


Web的设置很重要

这个里面设置的是项目文件的属性

4、然后是Artifacts里面的设置

尤其要注意这个里面的部署的目录，如果上一步骤中的WEB没有配置正确的话，这里也不会正确的，那么tomcat不会正常运行（404错误页面），但是XXXXXApplication可以正常加载出来，能够访问

5、除此之外，如果是用自己配置的tomcat来运行这个项目的话，有补救措施

注意啦，这里面多了一个SpringBootServletInitializer的继承实现，这样的话，就可以用自己配置的tomcat跑项目，如果不继承这个就不用写configure方法，但是就只能用IDEA自己的内置tomcat跑项目（不知道是不是tomcat）

6、然后说下springboot的框架搭建
Src的目录

这里面HelloController里面是controller的描写，尤其注意@RestController相当于@Controller和@ResponseBody的合体，今天在这里面捣鼓了半天，如果要返回一个页面，直接@Controller就OK了，这个类上面有无@RequestMapping（“/tengyun”）都可以，方法上面要有路径

FifthtestApplication里面就是main方法和继承SpringBootServletInitializer的@Overrides的configure方法，继承的这个方法是为了让自己配置的tomcat能跑起来

在applicaiton.properties中写上spring.thymeleaf.prefix=classpath:/templates/这样就可以返回templates文件夹中的页面了

Templates文件夹中存放你所需的page，应该可以改的
