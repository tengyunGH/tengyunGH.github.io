搭建网站步骤

1.注册github https://github.com ，创建相应的仓库
2.下载与你电脑匹配的git并安装 下载：https://git-scm.com/download 安装参考：https://www.cnblogs.com/smuxiaolei/p/7484678.html
3.将git与github绑定
4.安装node.js
5.搭建hexo，
6.绑定域名
7.写文章，推送

注册github https://github.com

创建一个仓库，仓库的名字一定是你的用户名.github.io

下载与你电脑匹配的git并安装 下载：https://git-scm.com/download 安装参考：https://www.cnblogs.com/smuxiaolei/p/7484678.html

landscape主题使用心得

没有系统性的学过ejs，但是大概理解还是可以的。
想自己写个主题的话，可以参考https://segmentfault.com/a/1190000006057336#articleHeader1 我觉得思路比较清晰

好，这里面最重要的是layout文件夹了，这是网站的布局文件，我的layout.ejs

整个网站用layout.ejs模板构建，这个模板的组成代表着网站页面的组成，也就是我每点击一个链接，页面的布局都按照layout.ejs来布局

明确一下：一个网页由head和body来组成，head一般是网页的一些属性，我们在浏览器上是看不见的，而渲染在浏览器上的是网页的body部分

1、head部分：第一行，引用head.ejs模板，这个是网站页面的html的<head></head>标签部分，可以在这个里面规划网站的标题、要引入的css文件等等

2、body部分：body部分由三部分组成（也就是我们看见的网页组成部分）：头-身-尾。
(1)头部：用了header.ejs模板渲染；
(2)身体：由一个section和一个sidebar.ejs模板渲染的工具栏部分（位置可以在主题配置文件中更改），section中的<%- body %>，这个里面的内容可以由archive、category、index、page、post、tag这六种模板布局来创建

我们在用hexo new [layout] “name”的时候，其实是在创建一个显示在网页body中的身体部位除了sidebar以外的部分，命令中layout可以省略，表示用默认布局，这个可以设置，在站点配置文件中有default_layout: post,这样表示默认布局文件是post，可以更改，改成其他五种，那么来说说这六类模板是干什么的

archive模板：对应archive.ejs文件，表示归档模板，与主题配置文件中的menu中的/archives链接相匹配，当点击这个链接时，网页body的身体中的section部分就由archive.ejs模板来布局

category模板：对应category.ejs文件，表示分类模板，我们hexo new category “某个东西”之后，这个东西就是一个category，所有这个东西的链接转至的网页，网页身体中的section部分就由category.ejs模板来布局

index模板：对应index.ejs文件，表示首页模板，与主题配置文件中的menu中的/链接相匹配，也和我们自己在hexo new index “name”的时候创建的东西对应的链接相匹配，当点击这类链接的时候，会转到网站的起始页面，当访问这个页面时，网页的body的身体中的section就由index.ejs来渲染

page模板：对应page.ejs文件，表示页面模板，当我们hexo new page “name” 就会在source文件夹下创建name文件夹，里面有一个index.md,可以在这里面写东西，要想显示这个页面，必须在主题配置文件的menu中添加相应的链接，这样，这个链接就会显示在网页body的头部中，点击那个链接，网页的body的身体中的section就由page.ejs来渲染内容，如果没有在menu中添加的话，是找不到这个链接的

post模板：当hexo new post “name” 的时候，会有一个name.md出现在source/_posts中，这个name.md对应的链接，点击他的时候，转到的网页的body的身体中的section由post.ejs来渲染。由于网站的首页模板，也就是index.ejs模板，index.ejs是这样的

将所有的post按照article.ejs模板来渲染并排列在首页中，所以我们在首页可以看到所有的post，当点击某一个具体的post的链接时，转到的网页的body的身体的section就会由post.ejs来渲染

tag模板 ：标签模板，和category类似，这里就不说了
body的身体部位小结：我觉得整个网站的布局是固定的，所有页面的不同之处仅仅是这个部分，网站的链接类型决定着转到的网页该部位的内容是什么样的，也就是说，链接类型决定使用什么模板来渲染这部分

(3)尾部：<%- partial('_partial/footer', null, {cache: !config.relative_link}) %>由footer模板渲染
