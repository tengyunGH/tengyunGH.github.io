Linux的文件系统

Linux下的文件系统

什么是文件系统？来理解一下

文件系统是基于被划分的存储设备上的一种文件的命名、存储、组织及读取的方法。

Linux文件系统的工作原理

文件系统通过为每个文件分配文件块的方式把数据存储在硬盘中。所以要维护每个文件的所有文件块的分配信息，这些信息也都存储在硬盘上。不同的文件系统，它的分配方式和读取文件块的方式是不一样的。
文件的块结构信息也称为meta-data（元信息），包括文件的权限、所有者、创建和访问和更改的时间等等这些与文件有关的信息。

Linux常见的文件系统分配策略

首先来了解一下磁盘的随机和顺序读写 参考自https://zhidao.baidu.com/question/1759018980722373228.html

分配策略有块分配和扩展分配
1、块分配
磁盘上的文件块根据需要分配给文件
优点：可以避免存储空间的浪费
缺点：当一个文件不断扩充的时候，可能会造成文件中文件块不连续，从而导致过多的磁盘寻道时间。读写文件时，可能要采用随机读写，而不能用顺序读取的方式，这样效率也会降低。
2、扩展分配
当文件被创建时，很多文件块同时被分配，当文件扩展时，也一次性分配很多块。文件的块结构信息meta-data在文件创建时写入，只要文件没有再分配文件块都不需要再写。当文件需要再次分配文件块的时候，再写入这个。
优点：文件会比较连续，若采用顺序存取的方式，读取的效率会很高
缺点：若一次性分配的块多了会浪费，若采用随机存取的方式，效率类似块分配

Linux的所有文件和目录以一个树状的结构组织构成文件系统。在硬盘上分区之后，还要在分区上创建文件系统，Linux常用的文件系统类型有：ext2/3/4、HFS、XFS、JFS、NFS、NTFS、FAT32等。Linux内核采用了虚拟文件系统(Virtual File System,VFS)的技术，因此Linux可以支持多种不同的文件系统类型。
我觉得就是，文件系统是一种规则，我们要按照这种规则来对硬盘访问、读写。

Linux中有哪些种类的文件

1、普通文件
2、目录文件
3、设备文件
4、链接文件
5、管道文件
6、套接口文件

来看一下Linux中的树状目录结构(既然这个目录结构已经确定，为什么还会有多种不同的文件系统呢？)
我觉得这个目录结构是Linux的标准，所有Linux支持的文件系统通过不同的方式来实现这个目录结构

/ -> 根分区
/boot -> 存放Linux系统启动文件，/boot/vmlinuz为linux的内核文件
/bin -> 可执行二进制文件的目录

/home -> 系统默认的用户目录，新增的用户宿主目录的父目录，比如用户user的主目录就是/home/user，可以用~user表示
/root -> 系统管理员root的主目录，系统第一个启动的分区是根目录/，所以最好将/root和根目录/放在同一个分区

/dev -> 存放设备文件，访问这个目录下的某个文件，相当于访问某个设备
/etc -> 系统管理和配置文件
/etc/rc.d -> 启动的配置文件和脚本

/lib -> 函数库保存位置，标准程序设计库，又叫动态链接共享库，作用类似windows里的.dll文件
/sbin -> 放置系统管理员root的可执行命令，一般用户只能查看
/tmp -> 公用的临时文件存储点，任何人都可访问，就不要在这里放重要的数据了

/mnt -> 系统提供这个目录是让用户临时挂载其他的文件系统。
/lost+found -> 这个目录平时是空的，系统非正常关机而留下“无家可归”的文件（windows下叫什么.chk）就在这里

/var -> 放置系统执行过程中经常变化的文件，比方说各种服务的日志文件，存放临时文件
/var/log：存放系统日志文件，/var/log/message：所有的登录文件存放目录，/var/spool/mail：邮件存放的目录，/var/run:程序或服务启动后，其PID存放在该目录下。建议单独分区，设置较大的磁盘空间

/proc -> 是系统内存的映射。可直接访问这个目录来获取系统信息。此目录的数据都在内存中，如系统核心，外部设备，网络状态，由于数据都存放于内存中，所以不占用磁盘空间

/usr -> 最庞大的目录，要用到的应用程序和文件几乎都在这个目录。其中包含：
/usr/X11R6 -> 存放X window的目录
/usr/bin -> 众多的应用程序
/usr/sbin -> 超级用户的一些管理程序
/usr/doc -> linux文档
/usr/include -> linux下开发和编译应用程序所需要的头文件
/usr/lib -> 常用的动态链接库和软件包的配置文件
/usr/man -> 帮助文档
/usr/src -> 源代码，linux内核的源代码就放在/usr/src/linux里
/usr/local/bin -> 本地增加的命令
/usr/local/lib -> 本地增加的库

SWAP -> 交换分区

使用文件系统的一般方法

1、在硬盘上创建分区或者逻辑卷：可使用fdisk命令创建分区，或者使用LVM相关的命令创建逻辑卷
2、在分区或者逻辑卷上建立文件系统：类似于windows的格式化操作
3、挂载文件系统到系统中：使用mount命令自己挂载，或者启动时自动挂载，这样子的话需要编辑“/etc/fstab”添加相应的配置行
4、卸载文件系统：对于可移动介质上的文件系统，使用完毕后可使用umount命令卸载文件系统

挂装文件系统的mount命令

格式：mount [可选项] [<分区设备名>] [<挂载点>]
这里面的可选项有：
-t<文件系统类型> 指定文件系统的类型
-r 使用只读方式挂载
-a 挂载/etc/fstab文件中记录的设备
-o iocharset=cp936 使挂载的设备可以显示中文文件名
-o loop 使用回送设备挂载ISO文件和映像文件
来看看实际例子
$ mount
$ mount -l
$ mount –guess /dev/sda3
# mount –t ext3 /dev/sdb1 /opt
# mount –t vfat /dev/hda6 /mnt/win
# mount –t vfat –o iocharset=cp936 /dev/hda8 /mnt/win2

卸载文件系统

格式：umount <分区设备名或挂载点>
粒子：umount /dev/hda6
umount /dev/sdb1
umount /opt

挂装/卸载文件系统要注意的

1、挂载点目录必须是存在的
2、应该在挂装目录的上级目录下进行挂装操作
3、不能在同一挂装点目录下挂装两个文件系统
4、当文件系统处于”busy”状态时不能进行卸载
5、系统是”busy”状态：文件系统上有打开的文件；某个进程的工作目录在此文件系统上；文件系统上面的缓存文件正在被使用
