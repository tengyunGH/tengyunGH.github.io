Java中的重写和重载

做题目的时候碰到重写和重载，以为自己搞清楚了，其实还有些问题，这里再来详细并清晰的记录一下

重写

子类和基类或者实现类和接口之间的关系，那么重写的时候，子类/实现类中重写的方法与基类/接口中的方法要遵循的规则：
1、方法名、参数列表一定要一致(一模一样)
2、访问权限，子类要比父类中的广 public > protected > friendly(省略) > private
3、返回值，子类要与父类中的相同或相容(此处的相容指的是，子类返回类型是父类返回类型的子类型，要是基本类型的话，就要相同了)
4、抛出的异常：子类抛出的异常一定是父类的异常的子集（子集的定义是：若集合A的任一元素都是B的元素，那么A是B的子集，所以A也是A本身的子集）
注：在子类重写父类方法或者实现类实现接口的方法时，可以使用@Override注解，来检查咱们是否符合重写规范

重载

重载指的是一个类中的方法间的关系，是相同的方法名，不同的其他东西，要遵循的规则：
1、方法名称一样
2、形参列表不一样
3、返回类型随便写，与他无关

写的不太清楚的地方，可以写个小程序实验一下

结果：
子类重写：2018
子类重载：java.lang.Object@15db9742
子类重写：子类重写
子类重载：2018
