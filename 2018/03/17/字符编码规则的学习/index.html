字符编码规则的学习

什么是字符编码？

计算机的世界是二进制的，8bit也就是我们常说的一个字节，一般用两位十六进制数表示。那么计算机如何存储从键盘或者从文件等其他地方读取的字符呢，字符编码就是解决这个问题。我们将字符编码，字符一一映射为一个数值，然后计算机将值存储，当要显示在屏幕时按规则映射为对应的字符。

字符有哪些

字符包括电脑键盘上的字符、中文、其他国家的文字

编码规则

1、ASCII码

全称： American Standard Code for Information Interchange，美国信息交换标准代码
规则：用8bit表示字符，其中，最高位为0，其他7位对应的值映射到每个特定的字符
组成：ASCII码共包含128个字符，其中有95个可显示字符，33个不可显示字符
ASCII码对照表： http://ascii.911cha.com/

2、EASCII码

由ASCII码扩充而来，最高位也加入表示行列，扩充出来的编码用来表示表格符号、计算符号、希腊字母等等，但是还是无法显示中文等一些语言。目前这种编码很少使用了。

3、ISO/IEC 8859 编码

也是由ASCII码扩充而来，这种编码，是将字符集分组，一共有15组，这样表示：ISO/IEC 8859-n 这里面n的取值是从1到16除掉12，所以一共是15组，例如ISO/IEC 8859-1 ISO/IEC 8859-2,其实可以理解为有15种编码规则了，所以一共可以表示128*15这么多个字符，但是在实际写一个东西的时候，只能运用一组规则，也就是说，不能同时支持多个组里面的字符编码。同时，这种字符集没有考虑亚洲文字。

4、Unicode码(统一码)

还有种编码叫 ISO 10646编码标准，Unicode和ISO 10646是两个独立的尝试创立单一字符集的组织各自开发的两种编码规则，一个是International Standard Organization(ISO)，一个是多语言软件制造商组成的统一码联盟。两者独立存在，却为创立一个单一编码表而协同工作。所以从大体上说，Unicode = ISO 10646。
世界上的所有字符数，大概有几万个。ISO 10646标准所定义的字符集称作为通用字符集Universal Character Set(UCS)。UCS-2表示用2个字节表示一个字符，UCS-4表示用4个字节表示一个字符。

Unicode编码规则只是定义了每个字符他的编码是多少，但是却没有定义字符对应的编码应该怎样存储在计算机中。如果Unicode统一规定，每个字符要用三或四个字节表示，但是常用的很多字符都是一字节就可以搞定的，那么对于存储来说，是极大的浪费，这样呢，也就形成了Unicode码的多种存储方式。
这种存储方式在互联网上有UTF-8、UTF-16、UTF-32，其中UTF-8是互联网上使用最广泛的Unicode编码的存储实现方式。注意，这里，UTF-8只是Unicode码存储的一种方式。
Unicode码的大小和存储的数据的大小(UTF-8)的对应关系：

比如说，某个字符的Unicode码表示为15678(1 0101 0110 0111 1000)，大小在上图的第四行，那么转换为UTF-8的存储数据之后，就是(11110000 10010101 10011001 10111000),就是按照数据的大小根据UTF-8的规则将二进制逐位补充进去，Unicode编码最多是能表示21bit的字符。

UTF-8编码有一个额外的好处，就是ASCII编码实际上可以被看成是UTF-8编码的一部分，所以，大量只支持ASCII编码的历史遗留软件可以在UTF-8编码下继续工作

我们常见的编码选项：
1、ANSI，对英文文件是ASCII编码，对简体中文是GB2312编码，繁体中文是Big5码
2、Unicode编码，是UCS-2编码方式，指直接用两个字节存入字符的Unicode码
3、UTF-8等等

中文字符编码标准

GB2312 GB13000 GBK GB18030
1、GB2312 1980年由中国自己制定，收录6763个汉字和682个其他符号；
2、GB13000 1993年是中国制定的等同于Unicode1.1的编码；
3、GBK 是微软对GB2312的扩展，利用未编码的空间，将GB13000中的全部汉字和字符也添加进来，还扩充了日语等一些文字，制定了GBK编码，所以GBK完全兼容GB2312，但是和GB13000比较，只是内容上GBK包含了GB13000
4、GB18030 2000年，国家出了GB18030标准，兼容GBK，取代GBK1.0，成为国家正式标准，该标准扩充了收录了汉字还有其他少数民族的文字。

ASCII => GB2312 => GBK => GB18030 后者兼容前者，在这些编码中，相同的字符的编码是同样的值，后面的标准支持更多的字符

GB2312、GBK、GB18030都属于双字节字符集

最终说一下本地编码，就是你的编辑器设置的编码方式，如果设置的是GBK，而用另一个地方的另一个编辑器打开，可能用的UTF-8的编码方式，这样就会解析成乱码了。如果说，你的文本里有两种不同的编码方式，那就要将他们都转换为Unicode码，才可以正确显示。
