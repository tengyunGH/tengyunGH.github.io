算法和算法的评价

什么是算法

算法是对问题的求解的一种描述。通俗说，就是解决一个问题的方案。

算法的5个重要特性：有穷性、确定性、可行性、输入、输出
1、有穷性：一个算法在执行有限的步数之后一定要停止
2、确定性：算法的每一条指令必须要有明确的含义。也就是说，相同的输入必须得到相同的输出
3、可行性：算法中的每个操作都必须是基本的运算执行有限次就可完成的
4、输入：一个算法有m(m >= 0)个输入
5、输出：一个算法有n(n >= 1)个输出

算法的目标：
1、正确性
2、可读性
3、健壮性：当输入非法数据时，算法能适当的反应或处理。
4、高效率和低存储

算法效率的度量

1、时间复杂度：T(n)
一个语句的频度是指该语句在算法中被重复执行的次数 。算法中所有语句的频度之和记为T(n)，T(n)是算法问题规模为n的函数。

来了解一个概念：算法的基本运算是指算法中最深层次的语句(多层for循环中的那个)。

而时间复杂度就是分析T(n)的数量级的，而T(n)的数量级又和算法的基本运算的频度f(n)是相同的数量级。所以时间复杂度的分析就转变为了对算法中的基本运算的频度的分析。

我们约定，时间复杂度等于f(n)中增长最快的一项将其系数置为1。举个例子，若f(n)=a*n³ + b*n² + c*n，那么时间复杂度T(n)=O(n³)（将a置为1了），其中的大O表示取这个运算式的数量级。

问题规模n应该好好理解一下，我觉得n应该就是理解为这个问题的范围，就是范围，意会吧。for(i = 0;i < n; i++)

计算时间复杂度时，我们选取最坏的情况去计算。

常见的时间复杂度有
O(1) < O(log₂n) < O(√n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)

2、空间复杂度：S(n)
首先什么是辅助空间？对数据进行操作的工作单元和存储一些为实现计算所需信息的这样的辅助的存储单元。
算法的空间复杂度并不是计算实际占用的空间，而是计算整个算法的辅助空间单元的个数，与问题的规模没有关系。算法的空间复杂度S(n)定义为该算法所耗费空间的数量级。S(n)=O(f(n))
若算法执行时所需要的辅助空间相对于输入数据量n而言是一个常数，则称这个算法的辅助空间为O(1);

递归算法的空间复杂度：递归深度N*每次递归所要的辅助空间， 如果每次递归所需的辅助空间是常数，则递归的空间复杂度是 O(N).
