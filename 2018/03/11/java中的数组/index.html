java中的数组

java中，数组是一种数据类型，属于引用数据类型。有java.lang.reflect.Array这个类，也有java.lang.Arrays这个类

首先，我觉得可以这么理解，一种数组(由存放的值的类型和长度共同决定)，相当于一个类，一个实际的数组 就像一个对象了

java.lang.reflect.Array类 ：创建数组、访问数组

Array类是一个final类，不可继承public final class Array {}
Array类中全部都是static或native方法

1、创建数组：在Array类中，数组的创建采用了newInstance方法(而不是构造方法),这个方法采用了反射的知识

所以我们实际上可以这么创建一个一维数组

但是我们平时是这么创建的

那么Array类里面的这些方法是用在，你不知道你想建立的数组成员的类型，那么采用Array中的方法，比如，你想建立一个和某个对象相同类型的数组，便可通过反射获取对象类型，从而创建数组。我猜想，我们平时用new方法创建数组的时候，编译器会调用Array的newInstance方法。

这里还有一个提一下：Array类的构造方法

网上有人说这句话 我还没太理解：“数组的实例化之所以不用构造函数的方法，是因为数组的类型其实是无数种。用构造方法来实例化不利于对不同类型数组的扩展”

2、设置和获取数组中的数值
设置数组用的是set方法，获取某个数组值用的是get方法

3、获取数组的长度length

小结：有人说，去除掉各种通过编译器简单化的写法后，真实的数组的创建和数据的操作就是通过Array类来完成的
参考自https://www.cnblogs.com/wuchangming/archive/2013/03/28/2986184.html

java.util.Arrays类

Arrays这个类呢，放在了util包里面，表示他是一个工具类，这个类中呢，都是一些静态方法，下面看下哪些方法比较常用
1、Arrays.sort(数组引用);或者Arrays.sort(数组引用, 开始排序位置, 结束排序位置);
功能：默认由小到大排序

sort采用快速排序方法，java8新增了parallelSort()方法，采用并行排序，就是将数组分成几个小的数组，每个子数组采用sort排序，最后合并结果。参考http://blog.csdn.net/u014115273/article/details/53318733 ，可以看下这两种方法的性能，当数据规模达到2的18次方时，并行排序的性能要远超过串行排序

2、Arrays.binarySearch(数组引用, 查找的元素) 或者Arrays.binarySearch(数组引用, 查找的元素, 开始查找的位置, 结束查找的位置)
功能：在某个数组中查找某元素的位置，以char类型的数组为例，这里数组类型可以是byte、short、int、long、float、double、类，

3、Arrays.toString(数组引用);
功能：将数组元素以[,,,,]的形式转换为字符串

如果，传入的数组是对象数组引用，那么在append的部分，拼接的是String.valueOf(对象元素)，也就是将所有对象的valueOf方法得到的字符串拼接，以逗号分离

4、Arrays.asList()方法：
功能：将这个数组转化为相应类型的一个List引用，指向ArrayList实例

注意此处的ArrayList并不是我们常用的那个，而是Arrays的一个内部类，这个类和我们常用的ArrayList同样都继承了AbstractList类，但是实现的接口不相同，AbstractList类又实现了List接口，因此，这样返回的一个对象赋给一个List引用，与外界我们常用的List list = new ArrayList<>();这种方式定义的对象结构是一样的，这种对象都只能调用List具有的方法，并且这个内部类实现的方法体与java.util.ArrayList中的方法体几乎一样（两者共同的方法），这样也保证了，内外的对象一致性

5、Arrays.equals(数组引用1, 数组引用2);
功能：比较两个数组的内容是否一致，注意只是内容

6、Arrays.copyOf(数组引用, 复制的长度)
功能：

java中数组的复制效率：System.arraycopy>clone>Arrays.copyOf>for循环

这里面还有个copyOfRange方法，可以将数组的指定范围部分复制给另一个数组

7、Arrays.fill(数组引用, 值)或者 Arrays.fill(数组引用, 从哪开始, 到哪结束 , 值)
功能：将数组的某一部分都赋值为某个数

关于数组

1.数组是java对象，但是不是Array类的对象，数组仅仅是个对象，我们在Array类中，用newInstance方法（实质是newArray方法）create一个数组，返回值类型是Object，所以我们create的数组是一个对象

2.new 语句执行：在堆区为数组分配内存空间-> 为数组中的每个元素初始化 -> 返回数组对象的引用
3.数组声明时要指定长度,且指定后便不可更改，数组引用属于变量，是可以更改的
4、在网上看到说《疯狂java讲义》中有这么一句话“如果想在java语言中实现这种可无限扩展的数组，则可以定义一个Object[]类型的数组，这个数组元素是Object类型，因此每个元素可以再次指向一个Object类型的数组”，也就是说，有个数组Object[],他的每个对象都是Object类型，所以每个对象又可以指向一个Object[],也就是数组的每个元素是一个数组，疯狂扩展，Infinite。
