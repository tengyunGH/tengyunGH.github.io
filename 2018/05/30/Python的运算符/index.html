Python的运算符

一、Python解释器有好几种

(1)CPython 用C语言开发，官方标准实现
(2)IPython 在CPython的基础之上在交互式方面得到增强
(3)Jython 为Java设计，把Python代码编译成Java字节码
(4)PyPy 是Python（2.7.13和3.5.5）的一种快速兼容的替代实现，速度快

二、算术运算符

1、+
2、-
3、*
4、/ 除 得到浮点数，就算能整除，也是浮点数

5、% 求余，模运算，如果有一个是浮点数，就算能余整数，结果也是浮点数，余浮点数，那就是浮点数。结果的符号和第二个操作数相同。当有一个为负数的时候，第二个操作数的整数倍大于且最接近第一个操作数的那个，然后减去第一个操作数。当操作数是浮点数，除了是0.5的其他都不精确。print(3.9%1.3) == 1.2999999999999998

6、** 幂次方 可以是小数 4**1.5 == 8

7、// 除,不要小数部分了,和Java的/相似，但是Java里面，1/0.1 == 10.0
但是Python里面
1//0.1 == 9.0
2//0.2 == 9.0
3//0.3 == 10.0
4//0.4 == 9.0
6//0.6 == 10.0

三、比较运算符

1、== 比较的是内容，只要内容一模一样，不管嵌套多少层地址不一样的都返回True。若元素内容一模一样，对于元素的顺序，String和Tuple和List中若顺序不一致，会返回False，Set、Dictionary只要元素内容一模一样就行，对元素的顺序没有任何要求。

2、!= 和==是相反的

3、> 数值就是大小 字符就是unicode码 字符串就是第一个不同的字符的unicode码比较，剩下的都不比了。 Tuple、List和字符串一样，只比较第一个不同的元素的大小。Dictionary不能参与这个比较的运算，><>=<=都不行。set的所有比较，除了元素全部相等的话，>= 和 <= 会返回True，其他的比较操作，其他的只要两者元素不等，都返回False

4、<
5、>=
6、<=

六、逻辑运算符

1、and 布尔与
2、or 布尔或
3、not 布尔非

那么这里就牵涉到了对象值和布尔值的等效了。

（1）和False等效的： Number的0和0.0和0+0j和0j还有-0、-0.0、-0j这样的、String的长度为0的空串””、Dictionary的内容为空的{}、Set内容为空的set()和set(“”)、Tuple的内容为空的()和(“”) 、List的[] 还有 None这个关键字，比如{1,2,3}.clear()的结果就是None
（2）Number的一切非0，Set的{“”}和非空的、Dictionary的{“”:””}还有其他的非空的、String长度不为0（包括空格）、List的[“”]还有一切非空的都等效于True
（3）可能还有一些，我还没有发现

七、成员运算符 是否在某个容器中

1、当用==比较后有一个元素成立，那就是in。记住双等号成立的：String、Tuple、List要求元素值和顺序完全一样，Set和Dictionary要求元素值一样就OK！
2、判断Dictionary里面的时候，是在判断Dictionary里面的key是否有一个，而Key的取值只能是不变类型：Number、String、Tuple，所以只能判断Number、String、Tuple这些的对象是否在Dictionary对象的Key里面。虽然Set也是元素不可变，但是感觉他的顺序没有固定。

1、in
2、not in
