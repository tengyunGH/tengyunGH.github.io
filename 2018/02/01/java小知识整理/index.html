java知识积累

java有些知识点不容易理解，还有些容易忘记，写篇博客，以后常来逛逛

访问控制

1、对类来说，有两种，一种是public，一种是缺省（friendly）。Public是所有类均可访问。缺省的话，只有当前类所在的包中的类可访问

2、对类中成员来说，有四种 public > protected > friendly > private。

3、Private只能在当前类中被成员方法访问，friendly(缺省)声明的，只能是当前包中的所有类都可访问。Protected不仅当前包中所有类，其他包中的子类也都可以访问，public是谁都可以访问。
注意访问范围：protected要比friendly的范围大

成员

包括 成员方法和成员变量

成员分为 静态成员和非静态成员

两者，由“静”这个字就可以看出，静态成员有常量的意味，不用具体的某个实例来决定他，当类定义的时候，他也已经被定义好了。非静态，就是说，他是随着环境变化的，当实例不同，他也就不同了。

封装意义

将数据和对数据的操作组合起来；
提供与外部的联系的方法时又尽可能的隐藏内部的实现细节；
使得设计和使用分离，使用者知道做什么，设计者知道怎样做。

继承原则

子类继承父类所有的成员变量（但子类成员方法不一定能访问），
子类继承父类所有除掉构造方法的成员方法，
子类可以增加、重写父类的成员，但是不能删除。
只有一个父亲，但是父亲却可以有多个后代。（单继承）

多态性（好复杂）

子类重定义
1）重定义父类成员变量，就隐藏了父类的成员变量；(如果是重载的话，就不会隐藏了)
2）重定义父类成员方法又包含两种：覆盖和重载；
<1>覆盖（重写）是说，参数列表（函数签名）相同，子类重新写父类（很多情况下是接口这种抽象的）中的那个方法。
<2>重载就是，一个类中，方法名相同，参数列表不同。由于子类继承了所有父类的方法，所以，如果子类的方法的参数列表不同，这就是重载，因为继承了，那个也是子类的方法，两者参数列表不同。

记住java中，重载要求参数列表不相同，若只有返回值相同，是会报错的。
还有java中的注解只有@Override，没有所谓的@Overload、@Overwrite,

使用@Override会帮你检查，你重写的方法和你要重写的那个方法方法名、参数列表是否一致。可以用在子类写父类，实现类写接口类当中。

今天发现了一个，如果，在父类的方法中，使用了父类的成员变量，那么子类继承这个方法后，子类在使用这个方法时，如果子类重定义了那个方法中的那个变量，那么此时，子类中有两个这样的变量，只是父类中的那个隐藏了，可以使用super调出，前提是子类能访问父类中的这个变量，子类对象调用父类的那个方法时，用的变量还是父类的；如果子类没有重定义，那就只能是使用父类的。总之，使用的变量都是父类的，要是子类能访问父类的那个变量，并且后来修改了，那就修改了。

接口、抽象类、类 之间的关系

接口和她的实现类之间只有实现关系，没有继承关系，接口和接口之间可以继承，类和类（包括抽象类）之间是继承关系。只能extends一个，却可以implements多个。

抽象类和抽象方法

public abstract class chouxianglei() {}
public abstract string chouxiangfangfa();
1）抽象方法没有实现，只有声明；
2）抽象类通常包含抽象方法，也可以没有；
3）包含抽象方法的类，必须声明为抽象类；
4）继承抽象类的子类，若不能实现所有父类的抽象方法，那么该子类要被声明为抽象类，也就说，可以不用全部实现，但是要声明为抽象类；
5）构造方法、静态方法不能声明为抽象方法。
6）抽象类不能实例化

说下java中的数据类型，有两大类

基本数据类型和引用数据类型
基本数据类型有 数值类型和布尔类型
数值类型又有 整形 浮点型 字符类型
引用数据类型有 类 接口 数组
注意String是个类，还是个final类

抽象类和接口比较 参考自 http://www.importnew.com/12399.html

1）两者是不同的类型，一个是类，一个是接口
2）抽象类允许有部分实现，接口完全是方法声明
3）一个类只能extends一个抽象类和可以implements多个接口。一个接口可以extends多个接口。
4）一个类可以实现部分extends的抽象类，但是要实现implements的接口的所有方法
5）抽象类可以有构造方法，但是接口没有。抽象类的子类在调用构造方法时，会调用继承的抽象类的构造方法。
http://blog.csdn.net/yoamy/article/details/51274614
6）接口的修饰符只有public，抽象类都有。
7）抽象类可以有main方法，可以运行，接口没有main方法

抽象类和接口的使用时机

如果你拥有一些方法并且想让它们中的一些有默认实现，那么使用抽象类吧。
如果你想实现多重继承，那么你必须使用接口。由于Java不支持多继承，子类不能够继承多个类，但可以实现多个接口。因此你就可以使用接口来解决它。
如果基本功能在不断改变，那么就需要使用抽象类。如果不断改变基本功能并且使用接口，那么就需要改变所有实现了该接口的类。

final类和final方法 final变量 final常量

翻译过来就是最终类和最终方法 最终变量 字符常量
1）final类不能被继承，final方法不能被子类覆盖
2）final类中的方法都是final方法，final方法可以在非final类中，只是不能被子类覆盖。
3）抽象类的设计应该是希望由不同的子类来完成，所以抽象类不能是final类。
4）final修饰的成员变量，必须在构造函数中完成初始化，并且，一次赋值，终生不变。基本类型值那就是值不变了，引用类型的，那它所指的对象就不变了。
5）final常量就是符号常量（常量分直接常量和符号常量，直接常量是那些直接用的常量比如1，“abc”这样的）
所以，如果你希望一个类不被继承，那么就声明为final类。

java中的传值

java中的传值均为值传递
1、当传递8中基本数据类型的时候，将实参的值拷贝给形参变量
2、当传递String类型值的时候，形参是新建的变量，这个变量指向实参的数据区域，若在方法体内改变形参指向的数据区域，那么形参就指向了不同的对象，实参却不会变化，也就是说，传String的时候，会新建一个String变量，这两个变量互不相干，都指向相同的字符串
3、当传递对象的时候，形参和实参的值一样，都是那个对象的地址，若形参改变了地址（重新new赋值），就和实参断开了联系（数组也传递的是地址）

析构函数

java中的finalize方法是在Object中定义的protected方法，平时JVM会帮我们自动回收，所以平时不需要我们自己去调用finalize方法，如果真的需要的话，可以调用的。
java中的finalize方法没有实现自动递归， 所以在子类的finalize方法里的最后一句话要加上super.finalize();

父子类的静态 非静态 构造函数的执行顺序

当加载一个类的时候，只会初始化静态部分，当构造一个类的对象的时候，先初始化非静态部分，然后执行构造函数初始化参数

执行顺序：父类静态初始化->子类静态初始化-> 父类非静态初始化 -> 父类构造函数 -> 子类非静态初始化 -> 子类构造函数
